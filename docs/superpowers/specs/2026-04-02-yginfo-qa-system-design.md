**日期：** 2026-04-02

## 1. 系统定位

**一句话描述：** 多渠道接入的企业级智能客服系统，基于知识库文档提供多轮对话和主动推荐能力。

**核心价值：** 7×24小时即时响应客户咨询，降低人工客服成本，提升响应效率。

**目标用户：** 中小企业（10-100人）用于产品FAQ客服场景

## 3. 核心流程

### 3.1 用户提问处理流程

```
用户提问 → 意图识别 → 知识库检索 → LLM生成 → 主动推荐 → 返回回答
```

**意图识别：** 判断用户问题是知识库问答还是闲聊/无效输入

**知识库检索：**
- 向量相似度搜索（top-k=5）
- BM25关键词召回（top-k=10）
- 融合排序取top-3

**LLM生成：**
- Prompt包含：用户问题 + 检索到的知识片段 + 对话历史
- Token限制：输入最多8000 tokens（GPT-4o 128k上下文），输出最多1000 tokens
- 检索结果截断：单段最多300字，最多传入3段

**融合排序算法：**
- 使用 Reciprocal Rank Fusion (RRF)
- RRF_score = Σ(1/(k+rank_i))，其中k=60
- 向量检索rank与BM25检索rank融合后取top-3

**主动推荐：**
- 根据当前问题，生成2-3个相关问题推荐
- 推荐来源：知识库中高频关联问题

### 3.2 多轮对话设计

- 上下文窗口：最近5轮对话
- 会话ID：基于用户ID+渠道+日期生成，同一用户每次对话生成新session_id
- 会话持久化：支持同一用户跨session恢复历史上下文（通过user_id关联）
- 会话超时：30分钟无交互则结束会话
- 指代消解：LLM自动处理（通过Prompt实现）

## 4. 知识库管理

### 4.1 文档处理流程

```
上传文档 → 格式检测 → 正文提取 → 文本清洗 → 分段处理 → 向量化 → 入库
```

### 4.2 文档规格

| 项目 | 规格 |
|------|------|
| 支持格式 | PDF、Word（.doc/.docx） |
| 单文件限制 | 50MB |
| 分段大小 | 500字/段 |
| 段落重叠 | 前一段末尾50字与后一段开头50字重叠 |

### 4.3 知识库操作

| 操作 | 说明 |
|------|------|
| 上传文档 | 管理员后台上传 |
| 解析预览 | 管理员预览解析后的内容 |
| 索引重建 | 全量重建向量索引 |
| 增量更新 | 上传新文档时增量入链 |
| 删除文档 | 软删除，更新索引 |

## 5. 渠道集成

### 5.1 优先级规划

| 渠道 | 接入方式 | MVP支持 | 后续迭代 |
|------|----------|---------|----------|
| 网页Widget | JS SDK嵌入 | ✅ | - |
| 企业微信 | 企微客服消息API | ✅ | - |
| 钉钉 | 钉钉机器人webhook | ❌ | 后续迭代 |
| 飞书 | 飞书机器人webhook | ❌ | 后续迭代 |

### 5.2 消息协议

统一使用内部MessageProtocol格式：
```json
{
  "channel": "webchat|wechat|dingtalk|feishu",
  "user_id": "string",
  "session_id": "string",
  "message_type": "text|image",
  "content": "string",
  "timestamp": "ISO8601"
}
```

## 6. 数据安全

| 安全措施 | 说明 |
|----------|------|
| 传输加密 | HTTPS/TLS 1.2+ |
| 存储加密 | AES-256（敏感字段） |
| 多租户隔离 | 应用层查询强制带tenant_id过滤，向量库通过namespace隔离 |
| 敏感词过滤 | 可配置关键词黑名单 |
| 对话留存 | 默认30天，可配置 |

## 7. MVP范围

### 7.1 上线标准

- [ ] 网页Widget可嵌入任意网站
- [ ] 企业微信客服消息可收发
- [ ] FAQ问题回答准确率≥80%（基于测试集）
- [ ] 管理员可上传和管理知识库文档
- [ ] 支持多轮对话（5轮以内）

### 7.2 排除项（后续迭代）

- 复杂多轮对话场景（超过5轮）
- 钉钉/飞书渠道接入
- 数据分析报表
- 多语言支持
- 人工客服转接
